This is the getting started guide for development. If you're interested in benchmarking please visit the getting started guide for benchmarking.

  1. Get TFB
  2. Fork the TFB project to your own repository.
  3. Clone your TFB repository to your local environment to get started.
  4. Set up a development environment
  5. Typically a TFB environment is set up by using a remote server dedicated to this project or by using a virtual machine. Because you're getting set up for development, you can use an environment with a single computer running the database, the load generation system, and the web framework.
  6. Get started quickly and set up your development environment with vagrant, or
  7. Click here for the guide to get your development environment set up. Note: We provide scripts for configuring a Linux development environment using either Virtualbox or Amazon EC2, and are actively searching for help adding Windows.
  8. Enable Travis-CI
  9. Enable Travis-CI on your project fork, so that any commits you send to Github are automatically verified for correctness (e.g. meeting all benchmark requirements).
  10. Configure Your Environment
  11. Create a benchmark.cfg file, which is the benchmark.cfg.example with your specific configurations.
  12. Run a Test
  13. toolset/run-tests.py --install server --mode verify --test ${your-test-here} Note: Only include "--install server" if you're running the test for the first time.
  14. Clean Up
  15. Remove the /FrameworkBenchmarks/results directory if you'd like to run the same test again.
  16. Remove the contents of /FrameworkBenchmarks/installs/ if you'd like to re-install the framework dependencies.
